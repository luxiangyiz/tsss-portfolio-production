### 阶段 3：生产容器

#### RAG API Dockerfile (`rag-api/Dockerfile`)
- 基础镜像: `python:3.13-slim`
- 非 root 用户运行（`appuser`）
- 固定启动: `rag_app.public_main:app`（不开放管理/入库/内部接口）
- 健康检查: `curl /api/rag/public/health`
- 多 Worker: 2 个 uvicorn worker

#### Docker Compose (`deploy/docker-compose.production.yml`)

| 服务 | 镜像 | 端口策略 | 健康检查 | 资源限制 |
|------|------|----------|----------|----------|
| Nginx | nginx:1.27-alpine | 80/443 公网 | ✅ | 无 |
| WordPress | wordpress:6.7-php8.3-fpm | 内网 | ✅ | 无 |
| MySQL | mysql:8.0 | 内网(3306) | ✅ | InnoDB 128MB |
| RAG API | 自构建 | 内网(8000) | ✅ | 内存 512MB |
| Qdrant | qdrant:v1.13 | 内网(6333/6334) | ✅ | 内存 256MB |

- 5 个命名持久化卷（mysql/wordpress/qdrant/nginx_logs/rag_logs）
- WordPress 主题/插件只读挂载
- 安全组策略：仅 Nginx 暴露公网端口

#### Nginx 配置 (`deploy/nginx/site.conf`)
- HTTP → HTTPS 301 重定向（备案后生效）
- TLS 1.2/1.3，ACME 验证路径预留
- RAG API 限流: 10 rpm，burst 5
- 全站限流: 60 rpm
- 安全响应头: X-Frame-Options、X-Content-Type-Options、XSS-Protection、HSTS、Referrer-Policy、Permissions-Policy
- WordPress 管理入口仅允许 localhost
